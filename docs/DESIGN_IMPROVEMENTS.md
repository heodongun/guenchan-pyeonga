# Homeground 서비스 개편 메모

## 1. 개요
- 단순 게시판 + AI 톤에서 **실제 동네 라이프 서비스 "Homeground"**로 리포지셔닝.
- 기능 포커스: 공간 예약, 모임 일정, 후기/질문 게시판을 한 흐름에 녹여 운영자와 이웃 모두가 바로 쓸 수 있게 정리.

## 2. 서비스 콘셉트
- **공간 예약**: 공유주방/공방/회의실 등 로컬 공간 소개와 예약 CTA를 자연스럽게 노출.
- **모임 일정**: 이번 주 클래스·모임 캘린더를 카드형으로 배치, 당일 일정은 히어로 카드에서 바로 확인.
- **이야기 아카이브**: 기존 게시글 데이터를 "이웃들의 이야기"로 재해석해 후기/질문/노하우 공간으로 전환.

## 3. 디자인 방향
- **톤앤매너**: 따뜻한 오프화이트 배경 + 러스트 오렌지(#f26b4f) + 그린(#2d6a4f) 포인트. 유리/네온 대신 생활형 서비스의 질감 강조.
- **타이포그래피**: `Noto Sans KR`로 교체해 한글 가독성과 제품 신뢰감 강화.
- **컴포넌트화**: `brand-card`, `brand-button`, `brand-button-ghost`, `brand-pill` 등을 정의해 전 화면의 일관된 룩앤필 확보.
- **레이아웃**: 히어로(추천 스폿·오늘 일정), 서비스 섹션(3개 핵심 축), 일정 섹션, 이야기 피드 순으로 정보 설득 → 참여 CTA 흐름 구성.

## 4. UX 업데이트
- **내비게이션**: 실제 존재하는 섹션 앵커(`서비스/모임/이야기`)로 교체, 비어있는 링크 제거.
- **카피 전면 교체**: "신호", "노드" 등 SF 용어 제거 → 로컬 서비스 맥락(예약, 일정, 후기) 중심 서술.
- **상태 처리**: 로딩/비어 있음/오류 상태 모두 브랜드 스타일의 카드로 정리해 이질감 제거.
- **글/댓글 작성 흐름**: 입력 필드·버튼을 새 팔레트로 통일하고, 로그인 누락 시 안내 문구와 로딩 상태 복구.

## 5. 화면별 변화 요약
- **메인 홈**: 새 히어로 + 추천 공간 리스트 + 오늘 일정 카드, 3개 핵심 서비스 블록, 주간 일정 그리드, 커뮤니티 이야기 카드형 피드로 재구성.
- **게시글 상세**: 헤더 메타(날짜/조회수)와 작성자 표시를 생활형 톤으로 수정, 댓글 트리 가독성 개선, 삭제/에러/로딩 상태 정리.
- **게시글 작성**: 안내 카피 및 필드 플레이스홀더를 운영/후기 맥락으로 변경, 로그인 누락 시 로딩 중단 처리.
- **로그인/회원가입**: toss 스타일 잔재 제거, 브랜드 카드/버튼/입력 스타일 적용.

## 6. 앞으로의 제안
- 공간/모임 데이터가 연동되면 예약 CTA와 일정 필터를 실제 API와 연결.
- 게시글 목록/댓글에 태그(공간, 모임, 질문 등) 필터 추가.
- 캘린더형 뷰와 예약 알림 구독 플로우 추가로 "동선 전체"를 서비스 안에서 완결.
